Architecture Decision Records

ADR-001 — Static Hosting: GitLab Pages over Vercel / Netlify / AWS
Status
Accepted
Context
Need a publicly accessible URL at $0 cost. No server budget. Must be verifiable by recruiters without any login.
Decision
Use GitLab Pages. The pages CI job uploads a public/ artifact — that's the entire deployment model.
Consequences
No cold starts. No runtime server. Deploy is atomic. GitLab provides free Let's Encrypt TLS for custom domains.
Alternatives considered
Vercel (requires account), Netlify (build limits), AWS S3+CloudFront (cost + complexity).
ADR-002 — Python + Jinja2 over Next.js
Status
Accepted
Context
Need to generate a static site with scraped data. Options: Next.js (SSG), Hugo, Jekyll, or Python.
Decision
Python + Jinja2. One entry point: python build.py. No npm, no node_modules, no webpack.
Consequences
Fastest debug loop. CI image is python:3.11-slim. Any change is: edit Python → rerun script → see result.
Alternatives considered
Next.js (npm dependency chain), Hugo (Go templating learning curve), Jekyll (Ruby runtime).
